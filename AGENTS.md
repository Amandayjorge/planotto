# Repository Guidelines

## Структура проекта
Приложение построено на Next.js (App Router) и хранит основное состояние в `localStorage`.

- `app/` — страницы и клиентская логика.
- `app/menu/` — недельный календарь меню.
- `app/shopping-list/` — список покупок.
- `app/pantry/` — кладовка.
- `app/recipes/` — список, создание и редактирование рецептов.
- `app/lib/` — общие утилиты (например, подсказки продуктов).
- `public/` — статические файлы.

Изменения вносите в соответствующий модуль, без дублирования логики между страницами.

## Команды разработки
- `npm run dev` — запуск локального сервера разработки (`http://localhost:3000`).
- `npm run build` — production-сборка и проверка корректности маршрутов/типов.
- `npm run start` — запуск собранной версии.
- `npm run lint` — проверка ESLint (Next.js + TypeScript).

Перед PR минимально запускайте: `npm run lint` и `npm run build`.

## Стиль кода и именование
- Язык: TypeScript + React functional components.
- Имена компонентов и типов: `PascalCase`.
- Переменные и функции: `camelCase`.
- Папки маршрутов: `kebab-case`, динамические сегменты — `[id]`.
- Предпочтительный стиль: короткие функции, явные интерфейсы, без скрытых преобразований данных.

## Тестирование
Отдельный тестовый раннер пока не стандартизирован. Базовый контроль качества:

1. `npm run lint`
2. `npm run build`
3. Ручная проверка затронутых экранов

При добавлении тестов размещайте их рядом с модулем (например, `app/recipes/__tests__/`) и в первую очередь покрывайте работу с `localStorage` и пользовательские сценарии (создание, редактирование, удаление).

## Коммиты и Pull Request
История Git в текущем окружении может быть недоступна, поэтому используйте единый формат:

- Коммит: короткий глагол + область (`pantry: add product search`).
- Один коммит — одна логическая задача.
- В PR указывайте: цель, измененные пути, шаги проверки, UI-скриншоты для визуальных изменений.

## Практические правила
- Не ломать существующие ключи `localStorage` без миграции.
- Новые поля добавлять обратно совместимо.
- Для текстов интерфейса использовать единый язык и одну кодировку (`UTF-8`).
